## Установка mdk с официального сайта forge

Для начала перейдите по ссылке: https://files.minecraftforge.net/net/minecraftforge/forge/index_1.20.1.html и скачайте forge mdk:
![img](https://github.com/35mt/Ru-Minecraft-1.20.1-forge-modding-tutorial/blob/main/Системное%20хранилище/Pasted%20image%2020260517211122.png)

По факту это и есть основа мода, однако там можно (и желательно) вырезать много лишнего, а так-же настроить сам мод.
## Базовая настройка файлов

Далее распакуйте скачанный архив в любую папку вне OneDrive, так как во время синхронизации могут повредиться файлы сборки, что очевидно не хорошо.
После того как вы распаковали архив в папку, смело можете удалять 4 файла:
- README.txt
- LICENSE.txt
- CREDITS.txt
- changelog.txt
![img](https://github.com/35mt/Ru-Minecraft-1.20.1-forge-modding-tutorial/blob/main/Системное%20хранилище/Pasted%20image%2020260517213013.png)

Эти файлы не несут никакого функционала в мод и нужны лишь для информирования о изменениях по версиям, благодарностях, лицензии и общей информации для разработчиков соответственно.
Удалять их совсем не обязательно, однако чтобы не мозолили глаза можно - непонятных файлов тут и так хватает.
Кстати о непонятных файлах - пройдёмся по порядку зачем нужен каждый из них:
1) gradle - папка нужна для установки gradle, если его ещё нет на пк (gradle это сборщик проекта - то, что преобразует ваш код и настройки в мод)
2) **src** -ключевая папка, в ней пишется весь код и хранятся текстуру, модели, локализация и т. д. - к ней я ещё вернусь позже
3) .gitattributes - настройки git
4) .gitignore - файлы которые будет игнорировать git для удобства и экономии места
5) build.gradle - зависимости и настройки сборки
6) **gradle.properties** - самый часто используемый файл в этом списке (не считая src) - в нём хранятся все визуальные настройки вашего мода (название, версия, описание и т. д.), а так-же пара важных настроек
7) gradlew - файл для сборки проекта из консоли на Unix системах
8) gradlew.bat - тоже самое только для windows
9) settings.gradle - ещё один файл с настройками сборщика

Кажется что файлов много, однако скорее всего после базовой настройки 1 раз вы не будете больше трогать никакие файлы кроме gradle.properties, а 
- gradle/ 
- gradlew 
- gradlew.bat
- .gitattributes
- .gitignore
вообще трогать не нужно.
## Первый запуск

Теперь вам нужно открыть свою IDE (в данном случае **IntelliJ**) и открыть в ней подготовленную папку. Это должно выглядеть примерно вот так:
![img|677](https://github.com/35mt/Ru-Minecraft-1.20.1-forge-modding-tutorial/blob/main/Системное%20хранилище/Pasted%20image%2020260124192027.png)

Далее вам будет предложено открыть проект в безопасном режиме или доверять ему. Выглядеть это должно примерно так:
![img](https://github.com/35mt/Ru-Minecraft-1.20.1-forge-modding-tutorial/blob/main/Системное%20хранилище/Pasted%20image%2020260124192533.png)

Здесь вам нужно нажать **Trust Project**, подтверждая, что вы готовы его открыть. Далее вам, возможно, будет предложено что-то докачать — вам нужно согласиться и установить все необходимые библиотеки и возможно ещё что-то.

!!! Некоторые загрузки могут длиться достаточно долго. Вы можете следить за процессом во вкладке build, которая выглядит примерно вот так:
![img](https://github.com/35mt/Ru-Minecraft-1.20.1-forge-modding-tutorial/blob/main/Системное%20хранилище/Pasted%20image%2020260124202000.png)

После всех установок у вас появится ещё 3 папки:
- .gradle
- .idea
- build

Первые 2 хранят индивидуальные настройки gradle и IDE - их трогать не нужно, а в папке **build/libs** будет хранится .jar файл (готовый мод), который можно создать позже, на этой странице: [Создание .jar файла с модом](Создание%20.jar%20файла%20с%20модом.md).
## Общие настройки

Для начала сделаем самое важное - заменим стандартное сопоставление на Parchment - это нужно, так как в стандартном сопоставлении имена большинства функций не читаемы, что сильно затрудняет разработку. Можете не волноваться - после сборки, мод будет не отличим от версии на стандартном сопоставлении. 

Для того чтобы установить Parchment перейдите на их официальный сайт: https://parchmentmc.org/docs/getting-started и следуйте инструкциям по установке. там всё предельно понятно даже без знания английского, единственное что отмечу: 
![img](https://github.com/35mt/Ru-Minecraft-1.20.1-forge-modding-tutorial/blob/main/Системное%20хранилище/Pasted%20image%2020260517223450.png)

Вместо этого пункта лучше перейдите в файл gradle.properties и укажите данные стабильные значения:
- mapping_channel=parchment  
- mapping_version=2023.06.26-1.20.1

Затем найдите подобную кнопку в правой верхней части экрана и нажмите на неё:
![img](https://github.com/35mt/Ru-Minecraft-1.20.1-forge-modding-tutorial/blob/main/Системное%20хранилище/Pasted%20image%2020260517224558.png)

Дождитесь сборки и всё - самое главное сделано.
## Индивидуальные настройки

Теперь для удобства дальнейшей разработки уберём всё лишнее:
Перейдите в папку src/main/java/com/example/examplemod, откройте файл ExampleMod и оставьте там только такую структуру удалив всё остальное:
![img](https://github.com/35mt/Ru-Minecraft-1.20.1-forge-modding-tutorial/blob/main/Системное%20хранилище/Pasted%20image%2020260517225802.png)

 А затем удалите файл **Config** в той-же папке.
Теперь, изменим название поля MODID на MOD_ID - это нужно для удобства, но не обязательно. Сделать это можно выделив название поля и нажав shift+f6. Это не влияет ни на что, кроме названия переменной.
Таким-же образом я бы посоветовал переименовать название главного класса мода

Теперь изменим значение этого поля на id мода - это **НЕ** отображаемое название в игре, этот код нужен для ориентации только внутри кода. В его названии не должно быть пробелов и заглавных букв, а так-же специальных символов - учитывайте это.
должно получиться примерно так:
![img](https://github.com/35mt/Ru-Minecraft-1.20.1-forge-modding-tutorial/blob/main/Системное%20хранилище/Pasted%20image%2020260518012324.png)

Допустим я введу название **myfirstmod**, теперь нужно перейти в gradle.properties, и в строке mod_id ввести такое-же значение - эти значения всегда должны совпадать, иначе мод просто не соберётся.

![img](https://github.com/35mt/Ru-Minecraft-1.20.1-forge-modding-tutorial/blob/main/Системное%20хранилище/Pasted%20image%2020260518013105.png)
Теперь во все поля со скрина можно вводить любые свои значения - они влияют как раз на отображение информации о моде в самой игре. поле отмеченное стрелкой пока не трогайте - оно должно указывать на относительный путь к главному файлу мода относительно папки java, по этому если вы решите изменить структуру папок то не забудьте изменить значение здесь.
![img](https://github.com/35mt/Ru-Minecraft-1.20.1-forge-modding-tutorial/blob/main/Системное%20хранилище/Pasted%20image%2020260518013724.png)

И ещё - версию мода принято записывать в формате:
**версия игры-версия мода**
Пример: **1.20.1-1.0.0**
это тоже не обязательно, но тоже повышает удобство для конечного пользователя
