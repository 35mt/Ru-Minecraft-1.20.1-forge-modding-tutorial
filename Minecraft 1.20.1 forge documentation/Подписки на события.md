Чтобы подписаться на событие нужно сделать 2 действия: 
1) создать класс обработчика событий (обычно 1 такой обработчик на мод) 
2) создать подписочный метод (в нём будет код, выполняемый при срабатывании указанного события) (сколько угодно на обработчик)

Я приведу пример, а затем объясню его:
```java
@Mod.EventBusSubscriber(modid = "Ваш mod_id", value = Dist.CLIENT)  
public class ClientEvents {  
    @SubscribeEvent  
    public static void onKeyInput(InputEvent.Key event) {  
        Minecraft mc = Minecraft.getInstance();  
        if (event.getKey() == InputConstants.KEY_N)  {  
            mc.player.sendSystemMessage(Component.literal("Нажата клавиша N"));  
        }  
    }  
}
```

## Класс обработчика
Здесь мы в первый двух сточках создаём класс обработчика событий, где:
```java
@Mod.EventBusSubscriber(modid = "Ваш mod_id", value = Dist.CLIENT)
```

Это аннотация для Forge, чтобы он вообще видел этот класс.
#### ВАЖНО

Что клиентские и серверные события нужно разграничивать, в ОСОБЕННОСТИ стоит отдельно указывать если событие клиентское через  **value = Dist.CLIENT**, иначе сервер просто упадёт при попытке обработки, так как на нём нет клиентских методов.

## Подписочный метод

Он создаётся на третьей и четвёртой строчке - 
- **@SubscribeEvent** добавляет этот метод в подписчики шины событий. без этой строчки метод будет просто висеть в классе и ничего не делать (естественно, если он ниоткуда больше не вызывается)
- **public static void onKeyInput(InputEvent.Key event) {** - сам метод который будет выполнятся в событии. аргумент который он принимает определяет тип события, при котором будет срабатывать. например в данном примере он будет вызываться по событию **InputEvent.Key** (нажатие клавиши)

В данном примере подписочный метод принимает в переменную **mc** клиент майнкрафта, затем проверяет - была ли нажата клавиша "N" и если да, выводит к игроку в чат "Нажата клавиша N"
