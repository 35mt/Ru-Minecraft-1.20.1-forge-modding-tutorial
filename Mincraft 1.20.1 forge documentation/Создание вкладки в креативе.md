для начала создайте файл класса java с названием ModCreativeModTabs и создайте там вот такой код:
```java 
import net.minecraft.core.registries.Registries;  
import net.minecraft.network.chat.Component;  
import net.minecraft.world.item.CreativeModeTab;  
import net.minecraft.world.item.ItemStack;  
import net.minecraft.world.item.Items;  
import net.minecraftforge.eventbus.api.IEventBus;  
import net.minecraftforge.registries.DeferredRegister;  
import net.minecraftforge.registries.RegistryObject;  
  
public class ModCreativeModTabs {  
    public static final DeferredRegister<CreativeModeTab> CREATIVE_MODE_TABS =  
            DeferredRegister.create(Registries.CREATIVE_MODE_TAB, MyFirstMod.MOD_ID);  
  
  
    public static void register(IEventBus eventBus) {  
        CREATIVE_MODE_TABS.register(eventBus);  
    }  
}
```
так-же замените **MyFirstMod** на свой главный класс мода
далее зайдите в главный файл мода и добавьте в конструктор строчку:
```java
ModCreativeModTabs.register(modEventBus);
```
как и в случае с созданием предмета данные действия нужны лишь для создания окружения и пока что не добавляют в игру ничего нового

теперь приступим непосредственно к созданию нового окна в творческом режиме.
для этого напишем вот такую строчку:
```java
public static final RegistryObject<CreativeModeTab> FIRST_TAB = CREATIVE_MODE_TABS.register("first_tab", () -> CreativeModeTab.builder()
		        .icon(() -> new ItemStack(ModItems.IMBA.get()))  
                .title(Component.translatable("creativetab.first_tab"))  
                .displayItems((pParameters, pOutput) -> {  
                    pOutput.accept(ModItems.ITEM1.get());  
                    pOutput.accept(ModItems.ITEM2.get());  
                    pOutput.accept(ModItems.ITEM3.get());  
                    pOutput.accept(Items.DIAMOND);  
                })  
                .build());
```
тут нужно многое объяснить, по этому я начну по порядку:
```java
public static final RegistryObject<CreativeModeTab> FIRST_TAB = CREATIVE_MODE_TABS.register("first_tab", () -> CreativeModeTab.builder()
```
эта часть создаёт непосредственно объект окна в творческом режиме и объявляет его код (в данном случае "first_tab"), а так-же создаёт поставщик который будет определять дальнейшие свойства вкладки. открывается цепочка свойств через метод 
```java
.builder()
```
а закрывается методом 
```java
.build()
```


теперь разберём сами свойства внутри данного билдера:
```java
.icon(() -> new ItemStack(ModItems.ITEM1.get()))  
```
этот метод добавляет иконку в виде одного из предметов которые добавляет мод - в данном случае этим предметом является "ITEM1"
```java
.title(Component.translatable("creativetab.first_tab"))
```
этот метод создаёт ключ объекта, который можно в дальнейшем использовать к примеру в локализации (подробнее дальше)
```java
.displayItems((pParameters, pOutput) -> {  
                    pOutput.accept(ModItems.ITEM1.get());  
                    pOutput.accept(ModItems.ITEM2.get());  
                    pOutput.accept(ModItems.ITEM3.get());  
                    pOutput.accept(Items.DIAMOND);  
                })  
```
этот метод самый главный - он отвечает за то, в каком порядке какие предметы будут находится в этой вкладке. на этом примере всё довольно наглядно, но я поясню - мы создаём поставщик, в который передаём 2 аргумента - первый нам пока не нужен, а второй является объектом, который имеет метод ".accept" при помощи которого мы и добавляем предмет на вкладку. как аргумент в **accept** нужно передавать экземпляр предмета, который нужно добавить на вкладку. предметы будут находится на вкладке в том порядке, в котором они указаны здесь, однако нельзя добавить больше одного одинакового предмета на вкладку - отображаться всё равно будет только 1. так-же можно добавить как предметы из вашего мода, так и из ванильного майнкрафта, как и показывает этот пример с алмазом.

хорошо, вкладка добавлена в игру, как и необходимые предметы добавлены в неё, однако у неё до сих пор нет локализации и называться она будет по именному коду, что не хорошо, так что прикрутим локализацию. 

снова откроем файл локализации который создали в прошлой записи по этому [[Локализация|гайду]] и добавим простую конструкцию типа:
```json
"creativetab.first_tab" : "Новая вкладка!"
```

теперь у вас есть полностью рабочая и готовая вкладка в креативе, однако если вы хотите добавить свои предметы на уже существующую вкладку, то просто перейдите в главный файл мода и найдите функцию "addCreative" в ней создайте подобную конструкцию:
```java
if (event.getTabKey() == CreativeModeTabs.INGREDIENTS) {  
    event.accept(ModItems.ITEM1.get());  
    event.accept(ModItems.ITEM2.get());  
}
```
думаю тут уже довольно очевидная аналогия с тем что я разобрал ранее. чтобы выбрать вкладку в конец которой будут добавлены ваши предметы просто измените **INGREDIENTS** на одну из этих ванильных вкладок:  **COMBAT, INGREDIENTS, BUILDING_BLOCKS, COLORED_BLOCKS, HOTBAR, FOOD_AND_DRINKS, FUNCTIONAL_BLOCKS, INVENTORY, NATURAL_BLOCKS, OP_BLOCKS, REDSTONE_BLOCKS, SEARCH, SPAWN_EGGS, TOOLS_AND_UTILITIES.**
