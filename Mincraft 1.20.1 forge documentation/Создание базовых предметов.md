возможность менять название этой папки под вопросом.

далее в этой папке создаём файл класса с названием "ModItems" и уже там пишем вот такой код:
```java 
import net.minecraft.world.item.Item;  
import net.minecraftforge.eventbus.api.IEventBus;  
import net.minecraftforge.registries.DeferredRegister;  
import net.minecraftforge.registries.ForgeRegistries;  
import net.minecraftforge.registries.RegistryObject;  
  
public class ModItems {  
    public static final DeferredRegister<Item> ITEMS =  
        DeferredRegister.create(ForgeRegistries.ITEMS, MyFirstMod.MOD_ID);  
      
  
    public static void register(IEventBus eventBus) {  
        ITEMS.register(eventBus);  
    }  
}
```
замените **MyFirstMod** на свой главный класс мода
а затем переходим в главный файл мода, и добавляем в конструктор вот такую строку:
```java
ModItems.register(modEventBus);
```

теперь мы наконец можем добавить первый предмет в игру, делается это всего лишь одной строчкой внутри класса ModItems:
```java
public static final RegistryObject<Item> ITEM1 = ITEMS.register("item1",  
        () -> new Item(new Item.Properties()));
```

ITEM1 - это будет просто переменная в коде указывающая на данный предмет, по этому название играет здесь роль исключительно для разработчика и никак не встречается в игре

указанное в кавычках "item1" - уже является кодом предмета в игре, но всё ещё не прямым названием которое увидит рядовой пользователь (важно что оно может содержать только строчные буквы, цифры и подчёркивания). однако, например, чтобы выдать себе предмет командой используется именно этот код. 

но как сделать читаемое для пользователя название? для начала создайте структуру папок в папке resources по этому [[Создание структуры папки resources|гайду]]. после создания структуры resources прочитайте так-же небольшой гайд по [[Локализация|локализации]]. после этого можно наконец создать пару - "ключ-значение" для локализации в формате 
"item.ваш mod_id.код предмета" : "название предмета". например в данном гайде это бы выглядело как:
````json
{
	"item.myfirstmod.item1" : "Предмет 1"
}
````
ну а для того чтобы добавить локализацию следующему предмету нужно просто поставить запятую и перейти на строку ниже:
```json
{
	"item.myfirstmod.item1" : "Предмет 1",
	"item.myfirstmod.item2" : "Предмет 2"
}
```


теперь предмет добавлен в игру, однако пока он без текстуры и модели, то бишь чёрно-фиолетовый куб в руках и инвентаре игрока. чтобы придать ему внешний вид перейдите в папку "item" внутри папки "models" и  создайте файл json с названием, соответствующим коду предмета, в случае этого гайда это будет "item1.json". затем зайдите в этот файл и создайте там вот такую структуру:
```json
{  
  "parent" : "item/generated",  
  "textures" : {  
    "layer0" : "myfirstmod:item/item1_picture"  
  }  
}
```
из этого всего нас пока-что интересует только "item1_picture" - так будет называться изображение, которое будет отображаться в руках и инвентаре игрока. у него может быть любое название строчными буквами и без пробелов, однако оно, очевидно, должно совпадать с названием самого изображения. теперь откройте папку "item" внутри папки "textures" и переместите туда своё изображение, в данном случае "item1_picture.png".

теперь всё готово и предмет добавлен в игру, однако его можно получить только через команду, а для того-чтобы добавить предмет в инвентарь креатива прочитайте данный [[Создание вкладки в креативе|гайд]] 

когда вы уже создали свой первый предмет можете переходить к созданию простого блока по этому гайду: [[Создание базовых блоков]]
